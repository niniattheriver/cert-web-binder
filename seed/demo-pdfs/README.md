# 데모 PDF (자체 생성 · 무저작권)

이 폴더의 PDF(2026년판 4부 + `2027/`의 2027 가상 개정판 4부)는 **전부 이 저장소의
`seed/pdf-gen/`이 자체 생성한 가상 콘텐츠**다.
실존 인증기관의 문항·서식·안내서, 실존 기관의 지침서와 무관한 창작물이며 저작권 문제가 없다.
(실제 인증 문항은 인증기관 저작물이므로 공개 저장소에 포함하지 않는다 — 설계서 §6.2.)

|---|---|---|

## 문항 PDF 서식 — 실물 서식(우수검사실 신임인증 심사점검표) 텍스트 골격

문항 PDF 2부는 실물 심사점검표의 텍스트 골격을 그대로 따른다(내용·분야명만 가상):

1. **표지** — `우수검사실 신임인증 심사점검표` / `[분야명]`(개인정보 보호 · 정보보안) / 발행기관(`가상인증원`) / `2026. 01`.
2. **개정 요약표** — `2026 년 개정판 변경내역 요약` / `(Summary of Changes)` / `[분야명]` 다음에 3구획:
   - `• 신규` → 열 헤더 `문항번호 사유` → `[번호] [사유]`(없으면 `-`)
   - `• 수정` → 열 헤더 `문항번호 수정유형` → `[번호] [수정유형]`(예: `50.020.010 문항 수정, 설명 수정, 배점 변경`)
   - `• 삭제` → 열 헤더 `문항번호 사유` → `[번호] [사유]`(없으면 `-`)
3. **문항분류체계** — `• 문항분류체계` / `핵심문항 : C` · `필요문항 : R` · `기본문항 : B`(파서 스킵 대상).
4. **목차** — `목 차` + 절 목록(파서 스킵 대상).
5. **본문 문항 블록**(반복):
   ```
   문항 [핵심C|필요R|기본B] [번호] [본문…]      ← 본문은 여러 줄로 이어질 수 있고 페이지 경계에 걸칠 수 있음
   배점
   (N) 아니오 [해당없음]                         ← N=배점(정수/소수), 해당없음 문항만 " 해당없음" 부가
   설명
   • [설명/불릿]
   ```
   본문 페이지에는 상단에 `쪽번호 + [분야명]` 러닝 헤더가 붙는다(파서가 무시).

- 유형(C/R/B)은 문항마다 배정: **핵심C 소수 · 필요R 일부 · 기본B 다수**
  (50: 5/9/16, 60: 5/10/10). 목록/상세의 문항 유형 배지 데모용.
- 파서 픽스처 케이스:
  - 본문이 페이지 경계에 걸치는 문항(50: `50.050.010`, `50.080.030` — 러닝 헤더를 사이에 두고 이어짐),
  - 개정표 `• 삭제`에만 있고 본문 블록에는 없는 삭제 문항(`50.030.050`),
  - 신규 문항(`50.090.010`), 소수 배점(`2.5`/`1.5`), 해당없음 문항(50: 5개, 60: 2개),
  - `배점 변경`(`50.020.010`)·`해당없음 유무변경`(`50.030.030`) 수정유형(변경 배지/재점검 트리거 대상).
- 텍스트 선택 가능(이미지 아님), 유니코드 NFC. `seed/pdf-gen/verify.mjs`로 추출 검증(파서 비의존, 자체 정규식).

## 2027 가상 개정판 (`2027/` — 연차 이월·자동 이관 데모용)

`seed/pdf-gen/content-2027.mjs`가 2026 콘텐츠를 원본으로 **변경분만 적용**해 만든 차기 연도판이다.
파일명은 2026년판과 동일하고 `2027/` 하위에만 생성된다. 2026년판 4부는 재생성하지 않는다(바이트 동일 유지).

| 파일 | 2026 → 2027 변경 | 쪽수 |
|---|---|---|
| `2027/가상지침서-개인정보처리지침.pdf` | 제12조 ① `지체 없이 파기한다` → `5일 이내에 파기한다` · 제4조 문구 완전 유지 · 제2027-1호, 2027. 1. 1. 시행 | 8 |
| `2027/가상지침서-정보보안운영지침.pdf` | 제13조(백업 및 복구) 문구 완전 유지 · 제18조 ② 경미 수정(`별도의 전문 교육` → `연 1회 이상 별도의 전문 교육`) · 제2027-2호 | 6 |
| `2027/가상문항-50-개인정보보호.pdf` | 신규 `50.090.020`(국외 이전 통지) · 수정 `50.010.010`(설명)·`50.040.020`(배점 3→4) · 삭제 `50.020.030` — 문항 30개 유지 | 10 |
| `2027/가상문항-60-정보보안.pdf` | 수정 `60.060.010`(백업 설명 문구)만 — 나머지 문항 동일 | 9 |

근거 연결(연차 이월) 데모 시나리오 — `seed/demo-anchors.json`의 인용문 기준:

- **자동 이관 대상**: 제4조(보호책임자) 인용문과 제13조(백업 및 복구) 인용문은 2027 개정판
  전문에 **정확히 1회씩 그대로 존재**한다(정확 일치 → 자동 이관).
- **검토 대상**: 제12조(파기) 인용문은 ① 문구 개정으로 2027 전문과 **정확 일치하지 않는다**
  (검토 목록으로 내려가는 케이스).

문항 ↔ 조문 호응표 변화(위 발췌표 대비):

- `50.020.030`(제3자 제공 동의·제공대장 — 제8조 호응) 삭제 → 해당 행 제외.
- `50.090.020`(국외 이전 통지·기록) 신규 → 지침서 제8조(제3자 제공)·제11조(이용내역의 통지) 취지와 호응.
- `50.040.020` 배점만 3→4로 변경(호응 조문 제16조(접근통제) 유지), `50.010.010`·`60.060.010`은
  설명 문구만 수정(호응 조문 제4조·제13조 유지).

검증: `node verify.mjs --y2027` — 2026과 동일한 서식 검증 + 위 인용문 3건(정확 일치 없음 / 1회 / 1회) 확인.

## 폰트 / 라이선스

- 임베드 폰트: **Noto Sans KR Regular/Bold** (`seed/fonts/`, OTF) — [notofonts/noto-cjk](https://github.com/notofonts/noto-cjk)
  `Sans/SubsetOTF/KR`. **SIL Open Font License 1.1** (`seed/fonts/OFL.txt`). PDF에는 pdfkit이 서브셋만 임베드한다.

## 재생성

```bash
cd seed/pdf-gen
npm install
npm run generate        # 문항 PDF 2부만 재생성(가상문항-*.pdf) — 지침서 PDF는 현행 유지
npm run verify          # pdfjs-dist(legacy)로 추출 검증(파서 비의존)
node generate.mjs --all # (선택) 지침서 PDF까지 전체 재생성

node generate.mjs --y2027 --all # 2027 가상 개정판 4부를 2027/ 에 생성(2026년판은 건드리지 않음)
node verify.mjs --y2027         # 2027/ 검증 + demo-anchors.json 인용문 유지/불일치 확인
```

- 생성 시 PDF 생성일을 고정(2026년판 `2026-01-01`, 2027년판 `2027-01-01`)하여 **재실행 시 바이트 동일**하다.
- 기본 `generate`는 문항 PDF만 덮어쓴다. 지침서 PDF(`가상지침서-*.pdf`)는 "현행 유지" 원칙에 따라 건드리지 않는다.
