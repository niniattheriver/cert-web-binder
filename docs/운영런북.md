# 운영 런북

내부망에 설치된 인증 근거자료 웹 바인더의 일상 운영 절차입니다.

> **비개발자 담당자용 요약**: 백업·점검·체크리스트는 화면에서 전부 할 수 있습니다 —
> **관리자로 로그인 → 오른쪽 위 사람 아이콘 → [운영 점검 (백업·점검)]**.
> 아래 문서의 `GET/POST /api/...` 표기는 IT 담당자·자동화용 참고이며 몰라도 됩니다.

## 서비스 재시작

- **Linux(systemd)**
  - 재시작: `sudo systemctl restart qaevidence`
  - 상태: `systemctl status qaevidence`
  - 로그: `journalctl -u qaevidence -e`
- **Windows(WinSW)**
  - 재시작: `qaevidence-service.exe restart` (중지 `stop` · 시작 `start` · 상태 `status`)
  - 로그: 설치 루트의 `qaevidence-service.out.log` / `qaevidence-service.err.log`
- **수동(점검용)**: 설치 루트에서 `NODE_ENV=production npm start`

## data/ 위치와 백업

모든 상태는 설치 루트의 **`data/`** 에 있습니다:

- `app.db` (+ `-wal` / `-shm`) — 문항·근거·채점·이력 등 전체 도메인 데이터(SQLite WAL).
- `files/` — 근거 PDF·이미지. **내용주소(sha256) 불변** 저장.
- `backups/` — 자동/즉시 백업 스냅샷.
- `session-secret.txt` · `initial-admin-password.txt`(최초 1회 — 관리자 비밀번호를 바꾸면 자동 삭제).

백업 방법:

- **전체 백업 ZIP (권장)** — 화면에서: **운영 점검 → [전체 백업 (PDF 포함)]** 버튼 →
  `data/backups/full-backup-….zip` 생성. **DB와 업로드한 PDF 원본이 전부** 들어가므로
  이 파일 하나를 USB 등 다른 장치에 복사해 두면 완전한 보관이 됩니다(파일이 많으면 몇 분).
  (IT 참고: `POST /api/admin/backup/full` — ZIP 내부는 `data/app.db`·`data/files/…` 구조라
  풀어서 그대로 복원할 수 있습니다. ZIP 형식 한계로 총 4GB를 넘으면 실패하며, 그때는 아래
  "오프라인 복사"를 사용하세요.)
- **빠른 백업 ZIP** — 화면에서: **운영 점검 → [빠른 백업 (DB만)]** 버튼 → `data/backups/backup-….zip`
  생성(DB 스냅샷 + 파일 목록 — **PDF 원본 본문은 미포함**). (IT 참고: `POST /api/admin/backup`)
- **자동 백업** — 앱이 매일 **03:00** 에 `data/backups/app-YYYYMMDD.db`(VACUUM INTO) 생성. **일 30개 + 월 12개** 보존. 별도 cron 설정 불필요(서비스가 상시 구동되어 있으면 됩니다). `pre-import-*.db`(가져오기 직전 스냅샷)와 수동 `backup-*.zip`/`full-backup-*.zip`은 자동 정리되지 않으므로 오래된 것은 확인 후 직접 지워 주세요.
- **오프라인 복사(회사 절차, 한 줄)** — `data` 폴더를 통째로 다른 장비·USB로 복사합니다.
  탐색기에서 설치 폴더의 **`data` 폴더를 복사 → 백업 장치에 붙여넣기** 하면 됩니다.
  - IT 참고(자동화): Windows `robocopy C:\qaevidence\data D:\backup\qaevidence-data /MIR` ·
    Linux `rsync -a --delete /opt/qaevidence/data/ /backup/qaevidence-data/`

> **⚠️ 실행 중 수동 복사 주의**: 서버가 실행 중일 때 `data` 폴더를 탐색기로 복사하면 DB 파일
> (`app.db`/`-wal`/`-shm`)이 복사 도중 바뀌어 서로 어긋난 사본이 될 수 있습니다. 가능하면
> **[전체 백업] 버튼**(실행 중에도 안전)을 쓰거나, 수동 복사는 서비스 중지 후에 하세요.

### 복원 (전체 백업 ZIP에서)

1. 서비스(또는 실행 창)를 **중지**합니다.
2. 전체 백업 ZIP을 임시 폴더에 **압축 해제**하면 안에 `data` 폴더가 있습니다.
3. 설치 루트의 기존 `data` 폴더 이름을 `data-망가짐-날짜` 처럼 바꿔 보관해 두고,
   압축에서 나온 `data` 폴더를 설치 루트로 **이동**합니다.
   (기존 `data` 를 재사용해 안에 덮어쓰는 경우에는 남아 있는 `app.db-wal`·`app.db-shm` 파일을
   반드시 삭제하세요 — 백업 스냅샷과 어긋난 이전 기록이 섞입니다.)
4. 서비스를 **시작**하고 로그인·문항·PDF 열람을 확인합니다.
5. `data/` 전체 복사본에서 복원하는 경우도 1·3·4 단계는 동일합니다(세션 시크릿·파일이 함께 따라옵니다).

## 다른 PC에서 접속이 안 될 때

서버 PC에서는 열리는데 동료 PC에서 안 열리면 순서대로 확인하세요:

1. **주소가 맞는지** — 동료 PC에서는 `localhost` 가 아니라 서버 PC의 주소로 접속해야 합니다.
   실행 창(검은 창)에 표시되는 **"같은 내부망에서 열기: http://…"** 주소를 쓰세요.
   (안 보이면 서버 PC에서 `ipconfig` → "IPv4 주소" 확인 → `http://그주소:8080`)
2. **Windows 방화벽 허용** — 첫 실행 때 [액세스 허용]을 누르지 않았다면 차단된 상태입니다.
   제어판 → Windows Defender 방화벽 → "앱 또는 기능 허용" 에서 `node.exe` 를 허용하세요.
3. **같은 네트워크인지** — 서버 PC와 동료 PC가 같은 내부망(같은 공유기/사내망)에 있어야 합니다.
4. **서버가 켜져 있는지** — 실행 창이 닫혔으면 서버도 꺼진 것입니다(상시 운영은 서비스 등록 참고).

## 사용자 추가

- `admin` 계정으로 로그인 → 관리 화면에서 사용자 생성.
- 역할: `admin`(관리) · `editor`(편집) · `viewer`(열람 전용 — 심사위원).
- viewer 임시계정은 만료일(`expires_at`)을 둘 수 있습니다(심사 종료 후 자동 비활성).
- 권한 규약: 인증 GET은 로그인 필수, 데이터 변경은 `editor` 이상.

## 관리 상태 신호

**운영 점검** 화면(관리자)이 아래 신호를 보여 줍니다. 빨간(경고) 신호의 의미와 조치:
(IT 참고: `GET /api/admin/status`)

| 신호 | 의미 | 조치 |
|------|------|------|
| **저장 공간 경고(빨강)** | `data/` 볼륨 여유공간 부족 | 오래된 `backups/` 를 다른 장비로 이관 후 정리, 볼륨 확장 |
| **백업 없음/오래됨(빨강)** | 최근 백업이 없거나 48시간보다 오래됨 | 서비스 상시 구동 여부·03:00 스케줄·디스크 확인 → **운영 점검**의 백업 버튼으로 즉시 생성 |
| **무결성 위반** | 자료 점검에서 실패 항목 존재 | 아래 "무결성 점검" 참조 |

## 무결성 점검

- 저장된 자료에 깨지거나 어긋난 곳이 없는지 검사합니다. 기동 직후 + 매주 자동 실행됩니다.
- 화면에서: **운영 점검 → [지금 점검 실행]** — 항목별 통과/위반이 표로 표시됩니다.
  (IT 참고: `GET /api/admin/integrity` · 즉시 실행 `POST /api/admin/integrity/run`)
- 위반 시: 화면을 캡처해 시스템 담당자에게 전달 → 위반 대상(`offenders`)을 편집 화면에서 정정.
  **자동 삭제·자동 수정은 없습니다.**

## 데모/초기화 (설치 검증용)

- 빈 `data/` 에서 데모 데이터를 시드: `npm run seed:demo`(admin/day1pass · 가상 분야 2개 · 문항 55개).
- 기존 데이터가 있으면 먼저 `data/` 를 `data.bak-<시각>/` 로 옮긴 뒤 실행하세요(안전장치가 덮어쓰기를 막습니다).

## 인증심사(실사) 전날 체크리스트

**운영 점검 화면 맨 아래에 이 목록이 체크리스트로 내장**되어 있어, 담당자는 화면에서
하나씩 체크하며 따라가면 됩니다. 전부 마우스 클릭으로 끝납니다:

- [ ] 운영 점검에서 **[전체 백업 (PDF 포함)]** 을 눌러 백업 파일 생성.
- [ ] 만들어진 **전체 백업 ZIP(또는 `data` 폴더 전체)을 USB 등 다른 장치에 복사**해 보관 (탐색기 복사→붙여넣기).
- [ ] 운영 점검에서 **[지금 점검 실행]** — 모든 항목 "통과" 확인.
- [ ] **사용자 계정 관리**에서 심사위원용 열람(viewer) 계정 발급 + 만료일 설정.
- [ ] 검색창에 문항번호를 입력해 **3초 안에 해당 문항이 열리는지** 연습.
- [ ] 분야 화면에서 **[엑셀 내보내기]** 와 인쇄가 되는지 확인.
- [ ] 서버 PC를 **재부팅한 뒤에도 접속되는지** 확인 (상시 운영 설정을 한 경우).
